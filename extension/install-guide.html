VSCodeにインストール

[FOLDER] VSCodeを起動

Visual Studio Code（VSCode）を起動してください。VSCodeがインストールされていない場合は、公式サイトからダウンロードしてください。

[KEY] コマンドパレットを開く

VSCodeでコマンドパレットを開きます。以下のキーボードショートカットを使用してください：

Windows / Linux: Ctrl + Shift + P
Mac: ⌘ + Shift + P

コマンドパレットに以下のテキストを入力してください：

[INFO] 検索のコツ

「vsix」または「拡張機能」と入力するだけで、該当するコマンドが表示されます。完全に入力する必要はありません。

[WARNING] 注意

「Extensions: Install from VSIX...」という英語のコマンドも同じ機能です。日本語版・英語版どちらを選んでも構いません。

候補が表示されたら、拡張機能: VSIX からのインストール...を選択してEnterを押してください。

使い方

[NOTE] コードを解析する（基本）

VIBECODINGでコードを解析する最も簡単な方法：

手順: 1. VSCodeで任意のコードファイルを開く 2. 解析したいコードを選択（マウスでドラッグまたはキーボードで選択） 3. 右クリック → 「VIBECODINGで解説」を選択 4. 解析が完了すると、サイドバーに解説が表示されます
右クリックメニュー
「VIBECODINGで解説」を選択
[OK] 解析完了

解説が生成されると、VIBECODINGパネルに以下の情報が表示されます：

  • コードの概要（サマリー）
  • 詳細な解説
  • 複雑度スコア
  • 重要な概念

[FAST] ファイル全体を解析する

ファイル全体を一度に解析することもできます：

手順: 1. 解析したいファイルを開く 2. Ctrl + Shift + P でコマンドパレットを開く 3. 「VIBECODING: 現在のファイルを解析」を選択 4. 解析が完了するまで待つ
[INFO] 大きなファイルの場合

ファイルが大きい場合、解析に時間がかかることがあります。タイムアウト設定を長めに設定することをお勧めします。

[FOLDER] フォルダを監視して自動解析

Claude Codeなどでコードを生成する際、特定のフォルダを監視して自動的に解析できます：

手順: 1. VIBECODINGパネルを開く 2. 「監視フォルダ」セクションで「+ フォルダを追加」をクリック 3. 監視したいプロジェクトフォルダを選択 4. フォルダ内のファイルが保存されると、自動的に解析されます
[WARNING] 注意

自動解析は設定で有効になっている場合のみ機能します。設定画面で「Vibecoding: Auto Analyze」がオンになっているか確認してください。

監視フォルダ設定
フォルダを追加すると、ファイル保存時に自動解析されます

[HISTORY] 解析履歴を確認

過去に解析したコードの履歴を確認できます：

手順: 1. VIBECODINGパネルを開く 2. 「最近の解析」セクションをスクロール 3. 見たい履歴項目をクリック 4. 解説が再表示されます
[INFO] 履歴の管理

履歴は最大10件まで保持されます。古い履歴は自動的に削除されます。

[RELOAD] パネルを更新

最新の情報を取得するには、パネルを更新できます：

VIBECODINGパネル右上の「[RELOAD] 更新」ボタンをクリック

更新すると、以下の情報が最新化されます：

  • 解析履歴
  • 最近の変更
  • 監視フォルダ一覧

トラブルシューティング

拡張機能が表示されない / アイコンが見つからない

解決策：

  1. VSCodeを完全に再起動してください（ウィンドウを閉じて再度開く）。
  2. 拡張機能ビュー（Ctrl + Shift + X）で「VIBECODING」を検索。
  3. 「無効」になっている場合は「有効にする」をクリック。
  4. それでも表示されない場合は、拡張機能を一度アンインストールして再インストールしてください。
「ログインが必要です」と表示される

原因： バックエンドAPIにログインしていない状態です。

解決策：

  1. VIBECODINGパネルを開く。
  2. 「ログイン」ボタンをクリック。
  3. メールアドレスとパスワードを入力してログイン。
  4. アカウントを持っていない場合は「新規登録」タブから作成。
「タイムアウトしました」というエラーが出る

原因： コードが長すぎる、またはサーバーの制限（Render無料プランは30秒）に達している可能性があります。

解決策：

  1. コードを分割して解析： ファイル全体ではなく、一部分ずつ選択して解析してください。
  2. タイムアウト設定を延長： 設定画面で「Vibecoding: Request Timeout」を長めに設定（最大60000ms = 1分）。ただし、サーバー側の制限は変わりません。
  3. 再試行： サーバーが初回起動中の場合、2回目以降は速くなります。もう一度実行してみてください。
「サーバーに接続できません」というエラーが出る

原因： バックエンドサーバーに接続できない状態です。

解決策：

  1. インターネット接続を確認： ネットワークに接続されているか確認してください。
  2. API URLを確認： 設定画面で「Vibecoding: Api Url」が正しいか確認（デフォルト: https://techstars.onrender.com）。
  3. サーバーの状態を確認： バックエンドサーバーが起動しているか確認してください（管理者に問い合わせ）。
解析結果が表示されない

解決策：

  1. VIBECODINGパネルが開いているか確認してください（左サイドバーのVCアイコンをクリック）。
  2. 「解説」セクションを展開してください（▼ボタンをクリック）。
  3. 開発者ツール（Ctrl + Shift + I）のConsoleタブでエラーを確認してください。
  4. パネル右上の「[RELOAD] 更新」ボタンをクリックして再読み込みしてください。

よくある質問（FAQ）

VIBECODINGの料金プランは？

料金プラン:

  • テックスターズAI契約者: 最初の2ヶ月は無料、3ヶ月目から月額5,500円（税込）
  • 一般ユーザー: 月額5,500円（税込）

すべてのプランで、AIによるコード解説、学習履歴管理、自動解析機能などの全機能をご利用いただけます。

Claude Codeとの連携方法は？

Claude Codeでコードを生成する際、生成先のプロジェクトフォルダをVIBECODINGの監視フォルダに追加してください。ファイルが保存されると自動的に解析が始まります。

オフラインでも使えますか？

いいえ、VIBECODINGはクラウドベースのAI解析を使用しているため、インターネット接続が必要です。

解析したコードのプライバシーは保護されますか？

はい、解析したコードはセキュアに送信され、APIサーバーで処理されます。詳細はプライバシーポリシーをご確認ください。

バグを見つけた場合、どこに報告すればいいですか？

GitHubリポジトリのIssuesセクションに報告してください。詳細な情報（エラーメッセージ、再現手順など）を含めていただけると助かります。
